UPDATE: Of course wikibooks has the most complete list of online collaborative editors haha - I can’t believe I didn’t come across this last time I was looking these up!

So I have a writing project that I need to work on with a group of people, and trying to determine the best way to work on it. We can use Word for development of content, but we probably want to publish with LaTeX so it’d be great if we could get the team on that. But I can’t imagine convincing the people I work with to install, configure, and learn LaTeX. We’re better off collaborating on googledocs and spending some time converting it to LaTeX code in one run, but that’s clunky and hard to update :/

Alternatives for collaborative writing (thanks stackexchange!)

  1. Googledocs is best for just writing text and copy into LaTeX when we’re done. Lol we’ll probably just do that.
  2. Share .TeX files using dropbox - limited by two things: only one person can edit at a time and all contributors must have TeX installed. Not happening.
  3. Share .TeX files using github or svn solves the one edit at a time problem, but wayy too much overhead for this situation.
  4. Online LaTeX collaborative writing site!

I can send my collaborators to a LaTeX website, they can probably deal with the simple LaTeX-code once we have a skeleton written. That would be beautiful! ;D


I’ve found two promising prospects

  1. WriteLatex is still in beta, but it’s super promising. This is the interface I think I want. Collaborative textpad on the left, and the pdf on the right. This one works off of CodeMirror as its text editor.
  2. ShareLatex looks great too, and more polished. This one works off of ace editor as its text editor.

I haven’t tested them (wanna try it with me? ;D), but I don’t think either is real-time in the way etherpad or googledocs are. I’d love for my LaTeX collaboration service to have real-time text input, real-time pdf generation, and support for LaTeX packages. I don’t think any option is quite there yet, but we’re off to a great start!

On the horizon, etherpad is developing a syntax highlighting plugin as we speak. Maybe it’ll do what I want soon? (but I’m not sure what codemirror/aceeditor has that I’d be missing out on, I haven’t checked them out yet)

There are two other projects I’m not quite as excited about:

  1. LatexLab which is based off of googledocs (ref) but it seems like it hasn’t been updated since 2010?
  2. ScribTex which seems totally developed but costs a yearly fee for any collaboration. Seems legit, but I don’t use LaTeX quite enough for this. (a review)
